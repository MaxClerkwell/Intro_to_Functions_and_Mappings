Die Abbildung als mathematisches Objekt

Abbildungen haben eine wohldefinierte Menge an Eigenschaften

Sie haben eine axiomatisch festgelegte Struktur

Eine Abbildung ist eine eindeutige Zuordnung von Elementen einer Menge zu Elementen einer anderen Menge.

Abbildungen dienen uns als Grundkonzept der Programmierung

Aufgabe früher Computer war es, Rechenmaschine zu sein

Interaktive Programme wie Betriebssysteme und GUIs kamen viel später

Um das Programmieren zu verstehen müssen wir den Computer verstehen

Um den Computer zu verstehen müssen wir Abbildungen verstehen

Definition

Eine Abbildung $f$ aus einer Menge $A$ in eine Menge $B$ ist eine Vorschrift, die jedem Element $a$ aus $A$ genau ein Element $b = f(a)$ aus $B$ zuordnet. Dabei nennt man $A=D(f)$ die Definitionsmenge, $f(A)$ das Bild, und $B=W(f)$ heißt Wertemenge.

Beispiele aus Mathematik und Informatik

Mathematische Funktionen:

Eine einfache quadratische Funktion:

f(x) = x^2

Eine lineare Funktion:

g(x) = 3x + 2

Beispiel aus der Informatik:

Programme als Abbildungen:

def square(x):
    return x * x

Hier entspricht die Funktion einer mathematischen Abbildung $f(x) = x^2$.

Programme führen Berechnungen durch, indem sie Eingaben in Ausgaben überführen.

Ein weiteres Beispiel für eine Funktion mit mehreren Eingaben:

def add(a, b):
    return a + b

Dies entspricht einer Abbildung $h: \mathbb{R}^2 \to \mathbb{R}$ mit $h(a, b) = a + b$.

Programme als Abbildungen

Programme lassen sich als Abbildungen von Eingaben auf Ausgaben verstehen.

Ein Programm verarbeitet eine Eingabe und gibt eine Ausgabe zurück:

def square(x):
    return x * x

Diese Funktion beschreibt eine mathematische Abbildung \( f: \mathbb{R} \to \mathbb{R} \) mit \( f(x) = x^2 \).

Programme können auch mehrere Eingaben verarbeiten:

def add(a, b):
    return a + b

Diese Funktion entspricht der Abbildung \( g: \mathbb{R}^2 \to \mathbb{R} \) mit \( g(a, b) = a + b \).

Einige Programme haben mehrere Ausgaben:

def add_and_multiply(a, b):
    return a + b, a * b

Hier wird eine Abbildung definiert, die ein Tupel \( (a+b, a*b) \) zurückgibt.

Programme sind oft Kombinationen von Abbildungen:

def f(x):
    return x + 3

def g(x):
    return 2 * x

def h(x):
    return f(g(x))

Hier ist \( h(x) = f(g(x)) \), also \( h(x) = 2x + 3 \).

Mengen

Definitionsbereich und Wertebereich

Was sind Mengen?

Eine Menge ist eine Sammlung wohldefinierter, unterscheidbarer Objekte.

Beispiele: Zahlenmengen (ℕ, ℤ, ℚ, ℝ), Zeichenmengen, Objekte in einer Datenbank.

Beispiele für Mengen

- Die Menge der natürlichen Zahlen: \( \mathbb{N} = \{ 1, 2, 3, ... \} \)

- Die Menge der Buchstaben im Alphabet: {A, B, C, ..., Z}

- Die Menge der gültigen Farben für eine Benutzeroberfläche: {rot, blau, grün}

Funktionen und Abbildungen

Eine Funktion $f: X \to Y$ ordnet jedem Element $x \in X $ genau ein Element $y \in Y$ zu.

Beispiel:

$f(x) = x^2, \quad X = \mathbb{R}, \quad Y = \mathbb{R}^+$

Reale Zahlen auf dem Computer

Auf Computern sind Zahlen nicht unendlich genau darstellbar.

Begrenzung durch:

  • Binärsystem: Darstellung mit 0 und 1
  • Speicherbreite: Float-Zahlen haben begrenzte Genauigkeit
  • Rundungsfehler: \( 0.1 + 0.2 \neq 0.3 \) in vielen Programmiersprachen

Interpretation einer Computerantwort

Computerausgaben sind oft codierte Werte:

  • Text als ASCII oder Unicode
  • Zahlen in Gleitkommadarstellung
  • Bilder als Pixelwerte in RGB oder CMYK

Auch hier gilt: Eine Abbildung von binären Daten auf interpretierbare Werte.

Eingabe – Verarbeitung – Ausgabe

Echter Wert wird auf Binärdaten abgebildet - kodiert

Binärwert wird durch Schaltung auf Binärwert abgebildet

Binärwert wird auf Darstellung abgebildet - dekodiert

Eigenschaften von Abbildungen

Injektivität

Eine Abbildung $ f: X \to Y $ heißt injektiv, wenn unterschiedliche Eingaben unterschiedliche Ausgaben liefern:

Injektivität

$f(x_1) = f(x_2) \Rightarrow x_1 = x_2$

Surjektivität

Eine Abbildung $ f: X \to Y $ heißt surjektiv, wenn jeder Wert in $ Y $ von mindestens einem $ x \in X $ getroffen wird:

Surjektivität

$\forall y \in Y, \exists x \in X: f(x) = y$

Bijektivität

Eine Abbildung ist bijektiv, wenn sie sowohl injektiv als auch surjektiv ist:

Bijektivität

$\forall y \in Y, \exists! x \in X: f(x) = y$

Anwendung dieser Eigenschaften in der Informatik

Injektivität notwendig für Speicheradressierung

Surjektivität zentral für Rundungen

Bijektivität essentiell für Kodierungen

Umkehrbarkeit von Abbildungen

Eine Abbildung $ f: X \to Y $ ist umkehrbar, wenn es eine Funktion $ g: Y \to X $ gibt, sodass:

$g(f(x)) = x \quad \text{und} \quad f(g(y)) = y$

Beispiel für eine umkehrbare Funktion:

$f(x) = x - 3, \quad f^{-1}(y) = y + 3$

Umkehrbarkeit von Abbildungen

Jede Bijektivität Abbildung ist umkehrbar

Nicht jede umkehrbare Abbildung ist bijektiv

Wann ist eine Funktion nicht umkehrbar?

Eine Funktion ist nicht umkehrbar, wenn sie nicht injektiv ist.

Beispiel:

$f(x) = x^2$

Da $ f(-2) = f(2) = 4 $, existiert keine eindeutige Umkehrfunktion.

Nützlichkeit der Unumkehrbarkeit

Einwegfunktionen bilden die Grundlage der asymmetrischen Verschlüsselung
